Windows Detected
Download Certificate

One install covers Chrome, Edge, and any other browser on this PC (they all use the same Windows certificate store). Firefox is the one exception — see the note at the bottom of this card.

1 Click the blue Download Certificate button above. Your browser will save a file called MOBIUS_Local_CA.crt — usually in your Downloads folder.
2 Open File Explorer (the yellow folder icon in the taskbar), click Downloads on the left, and find MOBIUS_Local_CA.crt.
3 Double-click the file. A small window titled Certificate will pop up. Click the button at the bottom labelled Install Certificate…
4 The Certificate Import Wizard opens. Under Store Location, click the Local Machine radio button (NOT “Current User” — this matters), then Next.
5 If Windows shows a blue User Account Control pop-up asking for permission, click Yes.
6 Click Place all certificates in the following store (the second radio button), then click Browse…
7 A list appears. Find and click Trusted Root Certification Authorities (you may need to scroll), then click OK.
8 Click Next, then Finish. Windows shows a red­dish Security Warning saying “Windows can't validate that the certificate is actually from MOBIUS Local CA.” This is expected — click Yes anyway. (You generated this CA yourself; the warning exists because Windows has never seen it before.)
9 Close every Chrome / Edge window on the computer (not just the current tab — the whole application). On the keyboard, Ctrl + Shift + Q quits Chrome. Then reopen and visit https://:8445 — the address bar should show a padlock, not “Not secure.”
Firefox users: Firefox keeps its own certificate list separate from Windows. Open Firefox, click the three-line menu in the top right, go to Settings → Privacy & Security, scroll down to Certificates, click View Certificates… → Authorities tab → Import…, select the MOBIUS_Local_CA.crt file, and check Trust this CA to identify websites.

Linux makes you pick the right install path depending on which browser you actually use. Chrome 105+ ignores the system trust store on Linux — the OS-level update-ca-certificates command only helps Firefox, curl, and a handful of other tools. For Chrome you need a separate policy file.

Google Chrome / Chromium (recommended)

One-shot install. The CA stays trusted across Chrome updates and survives server-cert rotations (only the CA is pinned).

1 Click Download Certificate above — saves MOBIUS_Local_CA.crt to ~/Downloads/.
2 Open a terminal and create the Chrome policy file (you'll need sudo): 
sudo mkdir -p /etc/opt/chrome/policies/managed

CA_B64=$(base64 -w0 < ~/Downloads/MOBIUS_Local_CA.crt)

sudo tee /etc/opt/chrome/policies/managed/mobius-home-ca.json > /dev/null <<EOF
{
  "CertificateManagementPolicyV1": {
    "trust_anchors": [
      { "type": "x509_root", "value": "$CA_B64" }
    ]
  }
}
EOF
3 Fully quit Chrome (not just close the window — pkill chrome if needed), then relaunch. Visit chrome://policy/ — you should see CertificateManagementPolicyV1 listed.
4 Open https://:8445 — padlock, not “Not secure.”

For Chromium instead of Chrome, the path is /etc/chromium/policies/managed/. Everything else is identical.

Firefox

Firefox uses its own trust store. Either the GUI or the command-line works; pick whichever is convenient.

1 GUI: Settings → Privacy & Security → View Certificates… → Authorities tab → Import… — pick MOBIUS_Local_CA.crt, check Trust this CA to identify websites.
2 CLI alternative (works for all NSS-backed apps):
certutil -d sql:$HOME/.pki/nssdb -A -t "C,," -n "MOBIUS Local CA" -i ~/Downloads/MOBIUS_Local_CA.crt

System tools (curl / wget / system Python)

Optional. Only run this if you call Smart Home from scripts or other CLI tools that consult the system trust store. 

sudo cp ~/Downloads/MOBIUS_Local_CA.crt /usr/local/share/ca-certificates/
sudo update-ca-certificates
Why Chrome needs its own thing: Since Chrome 105 (released August 2022), Chrome on Linux uses its own bundled Chrome Root Store and ignores both the OS trust store (/etc/ssl/certs) and the NSS database (~/.pki/nssdb). The Enterprise Policy file above is the supported way to add a private CA.
