Hubitat Device Capabilities Reference

Compiled February 2026 from official Hubitat Driver Capability List.
Source: docs2.hubitat.com/en/developer/driver/capability-list

A capability defines a contract: a set of commands (methods) and attributes (state values) that a driver implementing that capability must provide. The Maker API exposes these through the /devices/all and /devices/{id} endpoints.

Actuator Capabilities

Switch

AttributesTypeValues
switchENUMon, off

Commands: on(), off()

SwitchLevel

AttributesTypeUnit
levelNUMBER% (0–100)

Commands: setLevel(level, duration?)

level: 0–100, duration: transition time in seconds (optional)

ChangeLevel

AttributesTypeValues
No additional attributes

Commands: startLevelChange(direction), stopLevelChange()

direction: "up" or "down"

Commands: on(), off()

ColorControl

AttributesTypeDetails
colorSTRINGColor name or hex
hueNUMBER0–100
saturationNUMBER0–100
colorNameSTRINGNamed color (e.g., "Red")
RGBSTRINGHex RGB (e.g., "FF0000")

Commands: setColor(colorMap), setHue(hue), setSaturation(sat)

colorMap: {"hue":N, "saturation":N, "level":N} or {"hex":"RRGGBB"}
